# OpenTermix

*Читать на других языках: [English](README.md).*

OpenTermix — открытый лёгкий терминал в духе MobaXterm для Linux. Упор сделан на
быстродействие, минималистичный эргономичный интерфейс и нативную интеграцию с
вашей существующей настройкой SSH.

Написан на **Qt 6** (C++ в намеренно простом, «си-подобном» стиле),
использует [QTermWidget](https://github.com/lxqt/qtermwidget) для встроенного
терминала и [libssh](https://www.libssh.org/) для SFTP-браузера.

![OpenTermix](resources/opentermix.png)

## Возможности (MVP — SSH-ядро)

- Вкладочный встроенный терминал (локальная оболочка и SSH), запускает вашу
  системную оболочку.
- Разделение экрана: 1 / 2 / 4 терминала на вкладку.
- Мультиввод: одновременная трансляция нажатий во все открытые терминалы.
- Открепление вкладки в отдельное окно.
- Чтение и запись `~/.ssh/config`:
  - дерево сессий с группировкой в папки (хранится в комментариях
    `# OpenTermix-Group:` внутри каждого хоста, поэтому группировка не меняет
    порядок Host-блоков в конфиге),
  - графический диалог добавления/редактирования с пометкой и валидацией
    обязательных полей,
  - перед перезаписью конфига создаётся резервная копия с меткой времени.
- Встроенный SFTP-файловый браузер (перетащите файлы для загрузки, скачивание
  выделенного).
- Светлая/тёмная темы, настраиваемый шрифт терминала, сохранение раскладки окна.

Отложено на следующие этапы: VNC, встроенный X11-сервер, RDP/Telnet/Serial/Mosh,
менеджер SSH-туннелей, инструменты разработчика (редактор, генератор ключей,
diff, макросы), менеджер паролей, кросс-платформенная упаковка. Проброс X11
по-прежнему работает через `ssh -X`.

## Сборка (Linux)

### Зависимости

- CMake >= 3.18, компилятор с поддержкой C++17
- Qt 6 (Widgets)
- `qtermwidget6` (dev-пакет)
- `libssh` (dev-пакет)

На Debian (trixie/sid) / свежих Ubuntu:

```bash
sudo apt install cmake g++ pkg-config qt6-base-dev libqtermwidget-dev libutf8proc-dev libssh-dev
```

Примечания:
- Dev-пакет qtermwidget для Qt6 называется `libqtermwidget-dev` (он предоставляет
  pkg-config модуль `qtermwidget6`). В некоторых дистрибутивах — `libqtermwidget6-dev`.
- `libutf8proc-dev` нужен, потому что `qtermwidget6.pc` указывает его как
  зависимость.
- `pkg-config` обычно уже стоит на полноценной десктопной системе, но в
  минимальном окружении (например, свежем chroot от `debootstrap`) его может
  не быть — он нужен для `find_package(PkgConfig REQUIRED)` в `CMakeLists.txt`.

### Компиляция

```bash
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
./build/opentermix
```

### Сборка .deb-пакета

В каталоге `debian/` лежит стандартная конфигурация для `debhelper`. Помимо
зависимостей для сборки, установите инструменты пакетирования, затем соберите
пакет из корня репозитория:

```bash
sudo apt install debhelper cmake g++ pkg-config qt6-base-dev qt6-tools-dev qt6-l10n-tools \
                 libqtermwidget-dev libutf8proc-dev libssh-dev
dpkg-buildpackage -us -uc -b
```

Пакет `opentermix_<версия>_amd64.deb` (вместе с `.buildinfo`/`.changes`)
появится в родительском каталоге. Установка:

```bash
sudo dpkg -i ../opentermix_*_amd64.deb
```

Так собирать можно только прямо *на* поддерживаемой цели (см. ниже) — почему
`.deb`, собранный на одном дистрибутиве/релизе, обычно не ставится и не
запускается на другом, объясняется в разделе "Зависимости для работы".

#### Поддерживаемые цели

OpenTermix нужен Qt6-сборка `qtermwidget` (`libqtermwidget-dev` в Debian,
`libqtermwidget6-2-dev` в Ubuntu — `debian/control` объявляет оба варианта
как альтернативу). Такого пакета пока нет на всех deb-based релизах:

- **Debian**: trixie (13) и новее. В более старых релизах есть только Qt5-сборка.
- **Ubuntu**: 26.04 "Resolute" и новее (в 25.10 "Questing" тоже есть).
  **22.04 и 24.04 LTS не поддерживаются** — Ubuntu замораживает набор пакетов
  каждого релиза по снепшоту Debian за несколько месяцев до выхода, а Qt6-порт
  `qtermwidget` в Debian появился только в декабре 2024 — уже после заморозки
  обоих этих LTS. Ubuntu не бэкпортит новые версии библиотек в уже вышедший
  LTS, так что для них это вряд ли изменится.
- Производные (Mint, Pop!_OS, Zorin, ...) наследуют это от той базы
  Ubuntu/Debian, на которой они основаны.

#### Кросс-сборка под другую цель

Чтобы собрать `.deb` под цель, отличную от текущей машины, используйте
`scripts/build-deb.sh`: он поднимает одноразовый `debootstrap`-chroot под
нужный дистрибутив/релиз, собирает внутри него и копирует готовый `.deb`
обратно — так бинарник всегда слинкован с реальным Qt6/qtermwidget именно
той цели, а не с тем, что стоит локально.

```bash
sudo apt install debootstrap
bash scripts/build-deb.sh debian trixie
bash scripts/build-deb.sh ubuntu resolute
```

Скрипту нужен интерактивный `sudo` (для `debootstrap`/`mount`/`chroot`),
поэтому запускайте его в обычном терминале, а не через неинтерактивную
оболочку ИИ-ассистента.

### Зависимости для работы (runtime)

Установка `.deb` подтягивает:

- `libc6 (>= 2.34)`, `libgcc-s1 (>= 3.0)`, `libstdc++6 (>= 5)` — стандартные рантаймы C/C++
- `libqt6core6t64 (>= 6.8.2)`, `libqt6gui6 (>= 6.1.2)`, `libqt6widgets6 (>= 6.3.0)` — рантайм Qt 6
- `libqtermwidget6-2 (>= 2.1.0)` — встроенный виджет терминала
- `libssh-4 (>= 0.8.0)` — поддержка SSH/SFTP
- `qt6-svg-plugins` — плагин движка SVG-иконок, которым рисуются иконки
  тулбара (загружается в рантайме через `QIcon`, поэтому `dpkg-shlibdeps`
  не может обнаружить его автоматически — объявлен явно в `debian/control`)

Эти минимальные версии вычисляет `dpkg-shlibdeps` исходя из того, против какого
Qt6/qtermwidget собран пакет — то есть они зависят от машины сборки, а не от
целевой системы. Если собирать на Debian sid/unstable, итоговый `.deb`
потребует очень свежих версий библиотек и может не установиться на более
старых релизах (например, Debian stable) с ошибкой «unmet dependencies».

**Не исправляйте это вручную правкой номеров версий в `debian/control`.**
Qt встраивает в каждый бинарник версионный символ (`qt_version_tag`),
привязанный к конкретному релизу Qt, против которого он собран: бинарник,
слинкованный с Qt 6.10, не запустится с более старой `libqt6core6t64`,
независимо от того, что написано в метаданных пакета. Занижение версии в
`Depends` лишь заставит `dpkg` согласиться на установку — приложение затем
упадёт при запуске с ошибкой вида:

```
libQt6Core.so.6: version `Qt_6.10' not found
```

Чтобы собрать пакет под более старый дистрибутив, нужно собирать именно в
подходящем окружении (chroot через `debootstrap` или контейнер с нужным
релизом, либо сама целевая машина) — тогда бинарник реально будет слинкован
со старым Qt6/qtermwidget.

## Структура

```
src/
  app/        MainWindow: доки, меню, тема, настройки
  terminal/   TerminalWidget, TerminalGroup, TerminalArea (группы как в VS Code), MultiExec
  sessions/   Session, SshConfigParser, SessionTreeModel, SessionPanel, редактор
  sftp/       SftpClient (рабочий поток libssh), SftpBrowserWidget
resources/    QSS-темы + .qrc
translations/ файлы перевода Qt Linguist (.ts)
```

## Локализация

Интерфейс переводится через Qt Linguist. Переводы лежат в `translations/`,
компилируются в `.qm` и встраиваются под префикс ресурсов `:/i18n`; при запуске
загружается перевод, соответствующий системной локали.

Поставляемые локали: английский (исходный), русский
(`translations/opentermix_ru.ts`).

Как добавить новую локаль (например, французскую, `fr`):

1. Добавьте файл в `OPENTERMIX_TS_FILES` в `CMakeLists.txt`:
   `translations/opentermix_fr.ts`.
2. Сгенерируйте/обновите каталог строк из исходников:
   ```bash
   /usr/lib/qt6/bin/lupdate -recursive src -ts translations/opentermix_fr.ts
   ```
   (или после конфигурации: `cmake --build build --target update_translations`).
3. Переведите строки (в Qt Linguist или вручную).
4. Пересоберите — `.qm` скомпилируется и встроится автоматически.

## Примечания

- Аутентификация SFTP в этом MVP использует только ваш SSH-агент или ключи на
  диске. Парольная аутентификация запланирована вместе с менеджером паролей.
- Неизвестный ключ хоста сохраняется автоматически; *изменившийся* ключ прерывает
  подключение.
